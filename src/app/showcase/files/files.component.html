Component Anatomy Vue d'ensemble des composants de gestion de fichiers 
@ta/files-basic
├── ta-files-list [files] [canDeleteFile]
│   ├── (fileSelected)   → FileData & { index }
│   ├── (moreInformationSelected) → FileData
│   └── (fileDeleted)    → FileData
├── ta-documents-list [documentsIds] [actions] [readonly] [emptyMessage]
│   ├── (remove)         → string (document id)
│   └── (checkedFilesChanged) → InputUploadValue[]
├── ta-files-preview [initial: PreviewDocumentDto]
└── ta-files-preview-modal [open] [initial] (closeEvent)

@ta/files-basic
Files List — Images ta-files-list [files] [canDeleteFile] — Affiche une liste de fichiers image sous forme de cartes. Le 3e fichier simule l'état isLoading.
{{ this.canDeleteImageFiles() ? 'Masquer suppression' : 'Afficher suppression' }}
Files List — Documents ta-files-list avec type Document — PDF, Word, Excel. Formats détectés via fileExtension.
{{ this.canDeleteDocFiles() ? 'Masquer suppression' : 'Afficher suppression' }}
Files List — Empty State ta-files-list sans fichiers — affiche le composant ta-empty.
Documents List ta-documents-list [documentsIds] — Récupère les documents depuis l'API via TaDocumentsService. Passer un tableau vide affiche l'état vide. Requiert une API GraphQL configurée pour les IDs réels.
Empty (documentsIds=[])
Files Preview — Inline ta-files-preview [initial: PreviewDocumentDto] — Visionneuse intégrée. Détecte automatiquement le type de fichier (image, PDF, Word, Excel) via l'extension du nom.
Image (JPEG)
Document (PDF)
Files Preview — Modal Cliquer sur un fichier de la liste pour l'ouvrir dans ta-files-preview-modal.
@ta/files-extended
Files Upload ta-files-upload [features] [canSelectMultipleFiles] [showInActionButton] — Boutons d'upload. Les actions take-pic et upload-pic utilisent Capacitor (app mobile). upload-file utilise le FilePicker natif.
Images — take-pic + upload-pic (showInActionButton=true)
Documents — upload-file (showInActionButton=true)
Toutes features (showInActionButton=false)
